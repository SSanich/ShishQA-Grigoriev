
 Клиент серверная архитектура  -  представляет собой структуру работы продукта на всех ее стадиях.
 Самой распространненной, и,  вобщих чертах наглядной, будет трехзвенная архитектура представляющая собой клинта-сервер-базу данных. 
 На стороне клиента не хранятся данные и не выполняется их обработка. Клиент обращается к серверу и получает ответы. Таким образом сервер выполняет логику, БД хранение.
В простых (мало нагруженных) решениях может быть клиент-сервер. База данных находится на той-же машине, где реализуется логика. 
Для бизнес критичного ПО используются более сложные по содержанию архитектуры. Для работы под большими нагрузками, а так же обеспечения бесперебоиности  систем, используют кластеры серверов, баз данных. Клиент отправляет запрос на балансировщик , который определяет менее нагруженный сервер, чтобы передать ему запрос. Сервера баз данных тоже могут быть предсталвенны в кластере, которые так же равномерно  нагружаются. В случае выхода из строя одного из элементов кластера, нагрузка перераспределяется между другими.
Подобное разделение фукнций  позволяет:
- оптимизировать стоимость реализации проекта (когда логика лежит на сервере нет необходимости в мощных клиентских машинах)
- обеспечивать стабильность работы продукта (за счет кластеров серверов и/или кластеров баз данных)
- обеспечивать безопасность работы с данными, их передачи и хранения.
